建议学习路径

◆
按照"不熟悉 → 半懂 → 已掌握"的顺序，先攻克红点知识再补橙点，绿点可以跳过。每个阶段建议动手实践而不是只看文档。

第一周：认证机制（最大盲区）

用 EchoVerse 源码动手做：
① 用 curl/Postman 走一遍 注册→登录→带token访问→token过期 的完整流程
② 解码一个 JWT token（jwt.io 在线工具），看看里面存了什么
③ 理解 Session 和 JWT 的根本区别——"服务端存状态" vs "客户端存状态"

第二周：后端架构（中间件+控制器）

用 EchoVerse 源码实操：
① 在 auth.js 中加 console.log，观察中间件执行顺序
② 故意不带 token 调接口，跟踪请求在哪里被拦截
③ 在控制器中加断点或 log，观察 req.body / req.params / req.userId 的值

第三周：Postman 实战

① 创建一个 Collection，按"注册→登录→建专辑→收藏→评论"组织
② 学会用 Environment 变量存 token 和 songId
③ 写 Tests 脚本验证状态码和返回结构
④ 用 Collection Runner 批量执行

第四周：HTTP 协议深入 + 请求体格式

① 用 Postman 分别发送 JSON body 和 FormData body，观察 Content-Type 变化
② 测试 EchoVerse 的专辑上传接口（FormData + 文件）
③ 故意发错 Content-Type，观察后端报错

后续：CI/CD + 自动化 + 监控（了解即可）

① 了解 GitHub Actions 的基本概念
② 知道 Newman 可以在命令行跑 Postman 集合
③ 理解回归测试 vs 冒烟测试的区别
